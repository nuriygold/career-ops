## Slide 5. Architecture overview
**Subtitle:** A generalized, privacy-safe AI systems operating model

**Objective:** Show the major architectural layers in a sanitized way.

**Bullets:**
- User requests enter a structured runtime rather than a direct model call
- A model gateway handles routing and abstraction
- An execution layer supports tools, APIs, scripts, and workflows
- Observability layers provide logs, health checks, and status visibility
- Administrative controls support validation and troubleshooting

**Visual/layout direction:** Layered architecture stack with clear separation between layers.

**Speaker notes:**
This architecture is intentionally generalized, but it reflects a realistic pattern for agentic systems. Separating runtime, routing, execution, and observability makes the system easier to reason about and easier to improve over time.
